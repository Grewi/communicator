Любые данные переданные в сообщении могут содержать только структуру данных и их семантику. 
Данные определяющие внешний вид управляются на стороне клиента.
Например, в сообщении можно указать заголовок, изображение, важную часть строки, код, цитату, спойлер, список и т.д 
Но цвет, размер шрифта, отступы, фоны и т.д определяются стилями клиента


Набор возможных семантик зависит от типа сообщения. Изначально планируется три типа: Текст, MarkDown, Tags
Текст - Простой текст без разметки. Любая разметка должна отображаться как есть.
MarkDown - обработка разметки MarkDown
Tags - xml документ. Подобно HTM, но имена тегов отличаются и имеют только семантическое значение. Отсутствуют стили и скрипты
Сообщение может содержать вложения, изображение, медиа-галлерея, видео, файл в виде вложение.
Способы отображения вложений полностью зависят от клиента.
