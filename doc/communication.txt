Система подтверждения адекватности адреса.
Участники сети имеющие доверие со стороны других участников могут выдавать ссылки-подтверждения другим участникам.
Участники имеющие доверие, могут быть популярные сервисы, магазины, банки и т.д. своим клиентам могут предоставлять услугу
подтверждения адекватности. Предоставив ему уникальную ссылку. По этой ссылке он будет готов подтвердить любому другому участнику сети 
адекватность владельца этого адреса.

Один адрес может иметь любое количество таких ссылок. При запросе адекватности, может предоставить список ссылок.

По ссылке отдаётся факт подтверждения адекватности адреса. 

Необходимо прорааботать протокол где 
1) Клиент имеетвозможность сделать запрос на сервер о получении ссылки
2) Сервер может предложить ссылку 
3) Клиент соглашается/отказывается
4) Если клиент соглашается, сервер передаёт ссылку и делает себе отметку
5) Ссылка должна иметь срок годности, отсчёт идёт с момента принятия ссылки клиентом
6) По истечении срока годности ссылка удаляется или обновляется
7) Ссылка может быть удалена любой из сторон с уведомлением другой стороны.

Алгорим подтверждения
1) клиент обращается к серверу.
2) Сервер делает запрос к клиенту на получение списка 
3) Получив его ищет адреса  сервисов которым он готов доверять
4) Выбранным сервисам отсылает ссылку из списка
5) В ответ сервис должент вернуть адрес клиента.
6) Адреса должны совпасть
7) Если сервис не готов подтвердить адрес по любой причине, он должен вернуть ошибку 404 - страница не найдена

Подтверждением доверия является ответ с именем пользователя, хостом и публичным ключём шифрования.
