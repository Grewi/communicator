# Коммуникатор - система обмена сообщениями

Цель данного приложения, дать возможность обмена сообщениями не зависимо от архитектуры эвм, операционной системы,
языков программирования и среды, где необходимо его реализовать. 
Коммуникатор, это гибко настраиваемая и расширяемая система. Позволяет реализовать обмен сообщениями различными 
методами, образуя таки сущности, как личная переписка, групповые чаты, каналы, отдельные страницы с данными. А так же методы навигации и поиска. 
Любое взаимодействие вданной сети возможно только после взаимного подтверждения учётных записей. Соответствено, анонимные запросы практически невозможны. 

## Транспорт
Основным способом передачи данных является https протокол. Однако, для надёжной передачи данных могут использоваться 
сети на основе tor, i2p, smtp, а также других (в перспективе) протоколов. 

## Архитектура сети
Устройство взаимодействия напоминает почтовую переписку, когда каждый адрес, это учётная запись на сервере, а пользователи взаимодеййствуют с серверами через клиентскую программу. Т.е. Каждый пользователь взаимодействует только со своей учётной записью на сервере, которая в свою очередь взаимодействует с другим сервером. Цепочка отправки сообщения выглядит так:
Пользователь1 -> клиентская программа1 -> Сервер1 -> Сервер2 -> Клиентская программа2 -> Поьзователь2

Однако запросы, это не только отправка сообщения между двумя пользователями, ответом могут быть статичные или сгенертрованные страницы, которые в свою очередь могут образовывать произвольную иерархию. А также запросы предполагающие какое-то действие.
Например, подписка, запрос на переписку, отправка алерта и т.д.

## Подписки, Группы, Каналы
Подписка, это соглашеение возникшее в результате обмена запрашивающих и подтверждающих сообщений, в результате которых на одном из серверов создаётся запись о другой учётной записи которая наделяется правами чтения, редактирования, создания и удаления данных, в любой возможной комбинации которые образуют права этой учётной записи, а так же обязанность данного сервера оповещения этой учётной записи об изменеиях в материалах. На другом сервере, также создаётся аналогичная запись, позволяющая принимать уведомления от первого сервера.

Простое применение подписки, это канал. В этом случае с пользователем оформляется подписка только с правом чтения.
При публикации новых материалов всем подписчикам отправляется уведомление.

Более сложная форма подписки, это групповые чаты. В этом случае требуются более гибкие настройки прав, позволяющие читать, писать, редактировать и удалять сообщения.

## Коммуникационные сервера
Система взаимного доверия. Основана на серверах подтверждающих доверие дргих серверов. В качестве таких серверов могут выступать сервисы которые в процессе взаимодействия убедились в легитимности  такого адреса, либо специально созданные для этого сервера. Запись о легитимности должна присутствовать на обоих серверах: тот кто проверяет и тот кого проверяют. 
Список коммуникационных серверов должен максимально прозрачным для пользователя, процесс добавления может быть только с явным согласием. Также должен быть доступен процесс валидации записей.
Коммуникационные сервера могут участвовать в поиске по имени хоста и пользователя.
Запрос  к такому серверу, при положительном результате, должен вернуть имя пользователя, хост и публичный ключ
Запрос к искомому серверу должен подтвердить легитимность самого антибот сервера.

Антибот сервера могут использоваться для организации локальных сетей, в том числе и без доступа к глобальной сети.

Работа коммуникационных серверов напоминает серификационные центры, но с некоторыми существенными различаями:
    1. Один адрем может подтверждаться любым количеством коммуникацинных серверов
    2. Организация коммуникационного сервера доступна любому сообществу пользователей
    3. Процес максимально прозрачный для пользователя 

## Форматы сообщений
Коммуникатор позволяет использовать несколько способов форматирования сообщений. Однако главный принцип основан на строгом разделении: В тексте сообщения можно определять семанику отдельных частей текста, но не внешний вид.
Можно указать заголовки, важный текст, текст который нужно скопировать, ссылку, спойлер, список, код и т.д.
Внешнее отображение определяет только клиент, например: размер и цвет текста, отступы, фоны, поведение курсора и т.д.
