# HTTP заголовки

## Специфические заголовки

### X-Communicator-Version
Версия протокола коммуникатора используемая на сервере

### X-User-Name
Заголовок указывающий на имя пользователя коммуникатора которому направлено сообщение на данном сервере.

### X-Communicator-Search
Заколовок с адресом который требуется получить ответ от коммуникационного сервера.

### X-Type-Message
Тип сообщения.
    1. **connect** Запрос на установление контакта с правом отправки сообщения
    2. **alert** Быстрое текстовое сообщение
    3. **message** Обычное сообщение которое может быть письмом, сообщением в чат, комментарием и т.д.
    4. **file** Вложение или текст сообщения. Должен иметь размер, тип и id родительского сообщения
    5. **subscribe** Запрос на подписку 
    6. **communication** Обмен сообщениями связанными с получением/предоставлением ссылки или списка ссылок
    

### X-Id-Message
Уникальный id сообщения. 
При отправке файлов, они должны ссылаться на него.

### X-Communicator-To
Имя и домен пользователя которому направлено сообщение
user1@example.com

### X-Communicator-From
Имя и домен пользователя который отправил сообщение

### X-Transport-I2p
Адрес сервера в сети i2p уполномоченного принимать сообщения данного пользователя.
Используется в качестве резервного канала связи
Может иметь имя пользователя, хост, порт
user@g5lsqejlrfjcljodccykcacakylplotgu7exkk3vfsbzdrujw3sa.b32.i2p:12345

### X-Transport-Tor
Адрес сервера в сети TOR уполномоченного принимать сообщения данного пользователя.
Используется в качестве резервного канала связи.
Может иметь имя пользователя, хост, порт
user@g5lsqejlrfjcljodccykcacakylplotgu7exkk3vfsbzdrujw3sa.onion:12345

### X-Transport-Smtp
Адрес сервера smtp

## Стандартные заголовки

### Content-Language
Язык сообщения

### Accept
Ожидаемый тип содержимого. Используется в запросах

### Content-Type
Тип содержимого. 

### Content-Length
Длина содержимого
