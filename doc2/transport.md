# Транспорт
Основным транспортом является реализация mTLS с шифрованием ключами каждого сервера (Raw Public Keys) по RFC 7250

## Обмен ключами
Для первичного обмена ключами реализовать стандартное подключение с самоподписанными сертификатами.
Далее инициатор соединения отправляет список доверенных серверов
Принимающий сервер ищет общий доверенный сервер
Делает запрос на доверенный сервер и получает ключ инициатора
Принимающий сервер даёт ответ инициатору и сообщает ему общий доверенный сервер
Инициатор получив адрес доверенного сервера делает на него запрос и получает ключ принимающего серера
После этого сеанс заканчивается и дальнейшее соединение происходит по mTLS
