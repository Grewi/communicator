# Транспорт
Основным транспортом является реализация mTLS с шифрованием ключами каждого сервера (Raw Public Keys) по RFC 7250

## Обмен ключами
**Для первичного обмена ключами реализовать стандартное подключение с самоподписанными сертификатами.**
1. Далее инициатор соединения отправляет список доверенных серверов
2. Принимающий сервер ищет общий доверенный сервер
3. Делает запрос на доверенный сервер и получает ключ инициатора
4. Принимающий сервер даёт ответ инициатору и сообщает ему общий доверенный сервер
5. Инициатор получив адрес доверенного сервера делает на него запрос и получает ключ принимающего серера
6. После этого сеанс заканчивается и дальнейшее соединение происходит по mTLS
